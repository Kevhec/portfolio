<Row paddedRight>
  # Qué lo hace único

  Este proyecto nació de la necesidad de gestionar las finanzas de mi madre y ayudarla a ser más consciente de sus gastos. Inicialmente, busqué una aplicación existente que cumpliera con mis requisitos, pero descubrí que la mayoría eran de pago o requerían una suscripción premium para acceder a las funciones que necesitaba. Esto me llevó a tomar la decisión de construir mi propia aplicación, adaptada específicamente a mis necesidades y, al mismo tiempo, desafiando mis habilidades de desarrollo. Y así, el proyecto comenzó.
</Row>

<Row evenColumns inverted>
  <div>
    # Por qué estas tecnologías

    Para lograr las funcionalidades deseadas, sabía que debía implementar tanto el backend como el frontend. Por ello, elegí herramientas familiares como **Node.js** y **Express.js** como base para el backend, y **React** para el frontend.
    <br/>
    Enfocándome en el frontend, dado su carácter de dashboard, opté por **React** no solo por su idoneidad, sino también como una oportunidad para mejorar mis habilidades. Implementé un sistema de gestión de estado robusto utilizando la **Context API** y el hook `useReducer`, creando acciones modulares para cada contexto. Para el enrutamiento, seleccioné **React Router** debido a sus potentes capacidades y su API sencilla, lo que me permitió definir rutas claras y mantener un código modular y DRY-friendly. Para la UI, aproveché el sistema de componentes **Shadcn**, que me impresionó por su alto grado de personalización y control, especialmente al manejar envíos de formularios y validaciones gracias a su compatibilidad con `zod` y `react-hook-form`.
    <br/>
    Dado que mi experiencia con el desarrollo backend era limitada, consideré que **Express.js** era una buena opción para comenzar mi aprendizaje. Aproveché su sistema de enrutamiento integrado y su arquitectura basada en middlewares. En particular, utilicé middlewares para tareas como la verificación de *autenticación*, la validación de datos con **zod** y el control de acceso a recursos para operaciones en la base de datos.
    <br/>
    Para mi base de datos, elegí **Postgres SQL** debido a su alta popularidad y su amplia variedad de tipos de datos, y utilicé el **ORM Sequelize** para simplificar las operaciones con su completo conjunto de APIs. Además, implementé un sistema basado en cron con **node-cron** para gestionar transacciones y presupuestos basados en suscripciones, y por último adopté el patrón de arquitectura MVC (Modelo-Vista-Controlador) para la estructura de mi código.
